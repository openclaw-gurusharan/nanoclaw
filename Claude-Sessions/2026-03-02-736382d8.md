## Root Cause Analysis

### Incident 1 — `failed -> review_requested` blocked

Race condition in `worker-run-supervisor.ts`:

1. Primary worker container exits; completion invalid → repair spawned
2. During the ~2-min repair handoff gap, `reconcile()` sees `no container`
3. After `repairHandoffGraceMs` expires, supervisor auto-fails the run with
   reason `running_without_container`
4. Repair completes with valid output
5. `recoverWorkerRunForCompletionAccept` resets run to `running` (non-atomic)
6. Supervisor fires again BEFORE `updateWorkerRunStatus` runs → re-fails the run
7. `canTransitionWorkerRunStatus('failed', 'review_requested')` → `false`
8. Status update silently dropped → run stays `failed` forever

### Incident 2 — Andy greeting delayed

When Andy's container IS running, `queue.sendMessage(chatJid, formatted)`
succeeds (line 1557 of index.ts) — the greeting is piped to the active
container. But if Andy's container is a "no-output run" (stuck), the piped
greeting also sits unanswered.

`andyBusyAckLastSentMs` is declared (index.ts:113) and
`ANDY_BUSY_ACK_COOLDOWN_MS` is imported, but the ack logic was never wired
into the message loop. Dead code.

### Change 2 — Atomic completion accept

**File**: `src/db.ts`

Add new export `acceptWorkerRunCompletion(runId, data)` that wraps
everything in one SQLite transaction:

```ts
export function acceptWorkerRunCompletion(
  runId: string,
  data: {
    branch_name: string;
    pr_url?: string;
    commit_sha: string;
    files_changed: string[];
    test_summary?: string;
    risk_summary?: string;
    effective_session_id?: string | null;
    session_resume_status?: string | null;
    session_resume_error?: string | null;
  },
): { ok: boolean; recovered: boolean } {
  return db.transaction(() => {
    const run = getWorkerRun(runId);
    if (!run) return { ok: false, recovered: false };

    let recovered = false;
    if (run.status === 'failed' || run.status === 'failed_contract') {
      // Check if reason is recoverable
      let reason: string | null = null;
      try { reason = JSON.parse(run.error_details ?? '{}').reason ?? null; } catch {}
      if (!COMPLETION_RECOVERABLE_FAILURE_REASONS.has(reason ?? '')) {
        return { ok: false, recovered: false };
      }
      // Reset to running so transition below is valid
      db.prepare(`UPDATE worker_runs
        SET status = 'running', completed_at = NULL, phase = 'finalizing',
            active_container_name = NULL, no_container_since = NULL,
            expects_followup_container = 0, lease_expires_at = NULL,
            recovered_from_reason = ?
        WHERE run_id = ?`).run(reason, runId);
      recovered = true;
    }

    // Update completion fields
    db.prepare(`UPDATE worker_runs SET
      branch_name = ?, pr_url = ?, commit_sha = ?,
      files_changed = ?, test_summary = ?, risk_summary = ?,
      effective_session_id = ?, session_resume_status = ?,
      session_resume_error = ?, completed_at = COALESCE(completed_at, ?)
      WHERE run_id = ?`)
      .run(..., runId);

    // Transition to review_requested
    db.prepare(`UPDATE worker_runs
      SET status = 'review_requested', phase = 'terminal',
          active_container_name = NULL, no_container_since = NULL,
          expects_followup_container = 0, lease_expires_at = NULL
      WHERE run_id = ? AND status IN ('running', 'queued')`)
      .run(runId);

    return { ok: true, recovered };
  })();
}
```
